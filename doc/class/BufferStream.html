Class: BufferStream

Defined in: src/BufferStream.coffee
Inherits: stream.Writable

Overview

A buffer that grows in chunks, and allows waiting for a given number of bytes to be available.

Constructor Details

# (void) constructor(options = {})

Parameters:

  • options (Object) — can be as for stream, but also:

Options Hash: (options):

  • bsInit (Buffer) — initial buffer
  • bsGrowSize (Integer) — the number of bytes to grow when needed (default: 512)
  • bsStartEmpty (Boolean) — don't initialize with a growSize buffer (default: false)
  • bsStartEnded (Boolean) — after bsInit is appended, should we end the stream? if bsInit, defaults to true, else ignored
  • bsZero (Boolean) — whenever a new Buffer is added to the list, initialize it to zero (default: false)

# (void) wait(length, cb)

Wait for a given number of bytes to be available, then call the callback

Parameters:

  • length (Integer) — number of bytes to wait for
  • cb (function) — callback(error, buffer), where if the error is empty, the buffer will be exactly length bytes

Throws:

  • (Error) — invalid state, a second wait() while one was already pending
  • (Error) — invalid length
  • (Error) — no callback specified

# (Buffer) flatten()

Note: This does not fire any waiting callbacks, or remove bytes from the stream.

Smoosh everything into one buffer, with nothing left over This probably should never be called aside from internally or from the unit tests. May be changed to _flatten in the future.

Returns:

  • (Buffer) — the concatenated set of all bytes

# (Buffer) slice(start, end)

Note: This does not fire any waiting callbacks, or remove bytes from the stream.

Generate a buffer cropped by start and end. Negative indexes start from the end of the bytes currently in the stream.

Parameters:

  • start (Integer) — start offset (default: 0)
  • end (Integer) — end offset (default: @length)

Returns:

  • (Buffer) — the generated slice
